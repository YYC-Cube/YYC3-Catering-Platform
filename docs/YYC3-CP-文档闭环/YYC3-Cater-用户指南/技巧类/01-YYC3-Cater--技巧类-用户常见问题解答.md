## 1. 账号相关问题

### Q1: 如何注册账号？

**A**: 注册账号的步骤如下：

1. 访问平台地址：`https://catering.yyc3.com`
2. 点击"注册"按钮
3. 填写注册信息：
   - 用户名（6-20个字符）
   - 邮箱（有效邮箱地址）
   - 手机号（11位手机号）
   - 密码（8-20位，包含字母和数字）
   - 确认密码
4. 点击"注册"按钮
5. 查收验证邮件，点击验证链接
6. 验证成功后即可登录

**注意事项**：

- 用户名只能包含字母、数字和下划线
- 邮箱和手机号必须真实有效，用于找回密码
- 密码强度要求：至少包含一个大写字母、一个小写字母和一个数字

### Q2: 忘记密码怎么办？

**A**: 可以通过以下方式重置密码：

**方式一：邮箱重置**

1. 在登录页面点击"忘记密码"
2. 选择"邮箱重置"
3. 输入注册邮箱
4. 点击"发送验证码"
5. 查收邮件，获取验证码
6. 输入验证码
7. 设置新密码
8. 确认新密码
9. 点击"重置密码"按钮

**方式二：手机重置**

1. 在登录页面点击"忘记密码"
2. 选择"手机重置"
3. 输入绑定手机号
4. 点击"发送验证码"
5. 接收短信验证码
6. 输入验证码
7. 设置新密码
8. 确认新密码
9. 点击"重置密码"按钮

**密码要求**：

- 长度8-20位
- 包含字母和数字
- 不能与原密码相同
- 不能包含用户名

### Q3: 如何修改个人信息？

**A**: 修改个人信息的步骤如下：

1. 登录系统
2. 点击页面右上角的用户头像
3. 在下拉菜单中点击"个人中心"
4. 进入"个人信息"页面
5. 点击"编辑"按钮
6. 修改需要更新的信息：
   - 昵称
   - 邮箱
   - 手机号
   - 性别
   - 生日
   - 头像
7. 点击"保存"按钮
8. 系统提示保存成功

**注意事项**：

- 修改邮箱和手机号需要验证
- 头像上传支持 JPG、PNG、GIF 格式
- 建议头像尺寸 200x200 像素

### Q4: 如何绑定手机号？

**A**: 绑定手机号的步骤如下：

1. 进入"个人中心"页面
2. 点击"账号安全"标签页
3. 点击"绑定手机号"按钮
4. 输入手机号（11位）
5. 点击"获取验证码"按钮
6. 接收短信验证码（6位数字）
7. 输入验证码
8. 点击"确定"按钮
9. 系统提示绑定成功

**注意事项**：

- 手机号必须是11位有效手机号
- 验证码有效期为5分钟
- 每个手机号每天最多发送5次验证码

### Q5: 如何修改密码？

**A**: 修改密码的步骤如下：

1. 进入"个人中心"页面
2. 点击"账号安全"标签页
3. 点击"修改密码"按钮
4. 输入原密码
5. 输入新密码
6. 确认新密码
7. 点击"确定"按钮
8. 系统提示修改成功

**密码要求**：

- 长度8-20位
- 包含字母和数字
- 不能与原密码相同
- 不能包含用户名

**安全建议**：

- 定期修改密码（建议每3个月）
- 不要使用简单密码（如123456、password）
- 不要在多个平台使用相同密码

### Q6: 如何退出登录？

**A**: 退出登录有两种方式：

**方式一：正常退出**

1. 点击页面右上角的用户头像
2. 在下拉菜单中点击"退出登录"
3. 系统提示确认退出
4. 点击"确定"按钮
5. 系统清除登录状态，跳转到登录页面

**方式二：自动退出**
以下情况系统会自动退出：

- 登录状态超过7天未操作
- 在其他设备登录同一账号
- 账号被管理员禁用
- 连续5次密码输入错误

### Q7: 账号被禁用怎么办？

**A**: 如果账号被禁用，请按以下步骤处理：

1. 联系客服：400-123-4567
2. 提供以下信息：
   - 账号（邮箱或手机号）
   - 身份证明（如需要）
   - 被禁用原因说明
3. 等待客服审核
4. 审核通过后账号将自动解禁

**常见禁用原因**：

- 违反平台使用规范
- 恶意刷单
- 虚假交易
- 侵犯他人权益

## 2. 商品管理问题

### Q8: 如何添加商品？

**A**: 添加商品的步骤如下：

1. 点击左侧导航菜单的"商品管理"
2. 进入"商品列表"页面
3. 点击"添加商品"按钮
4. 填写商品信息：
   - 商品名称（必填）
   - 商品分类（必填）
   - 商品价格（必填）
   - 商品库存（必填）
   - 商品描述（选填）
5. 上传商品图片（选填）
6. 设置商品规格（选填）
7. 点击"保存"按钮
8. 系统提示添加成功

**商品信息填写规范**：

- 商品名称：2-200个字符，不能包含特殊字符
- 商品价格：必须大于0，保留两位小数
- 商品库存：必须大于等于0，整数
- 商品描述：0-5000个字符，支持富文本
- 商品图片：支持 JPG、PNG、GIF 格式，建议 800x800 像素

### Q9: 商品图片上传失败怎么办？

**A**: 商品图片上传失败的常见原因及解决方法：

**原因一：图片格式不支持**

- 解决方法：转换为 JPG、PNG、GIF 格式

**原因二：图片大小超过限制**

- 解决方法：压缩图片，建议不超过 5MB

**原因三：图片尺寸不符合要求**

- 解决方法：调整图片尺寸，建议 800x800 像素

**原因四：网络连接问题**

- 解决方法：检查网络连接，重新上传

**原因五：浏览器兼容性问题**

- 解决方法：更换浏览器（推荐使用 Chrome、Firefox）

**如果以上方法都无法解决，请联系客服**

### Q10: 如何批量上架商品？

**A**: 批量上架商品的步骤如下：

1. 进入"商品列表"页面
2. 勾选需要上架的商品
3. 点击"批量操作"按钮
4. 选择"批量上架"
5. 系统提示确认操作
6. 点击"确定"按钮
7. 系统提示操作成功

**注意事项**：

- 一次最多可以选择100个商品
- 批量操作后无法撤销
- 建议先检查商品信息是否完整

### Q11: 如何设置商品分类？

**A**: 设置商品分类的步骤如下：

1. 点击左侧导航菜单的"商品管理"
2. 点击"商品分类"子菜单
3. 进入"商品分类"页面
4. 点击"添加分类"按钮
5. 填写分类信息：
   - 分类名称（必填）
   - 父分类（选填）
   - 排序序号（选填）
   - 分类图标（选填）
6. 点击"保存"按钮
7. 系统提示添加成功

**分类层级建议**：

- 建议不超过3层
- 示例：
  ```
  中式餐饮
    ├── 面条类
    │   ├── 红烧牛肉面
    │   └── 番茄鸡蛋面
    └── 米饭类
        ├── 宫保鸡丁饭
        └── 红烧肉饭
  ```

### Q12: 商品库存不足怎么办？

**A**: 商品库存不足的处理方法：

**方法一：调整库存**

1. 进入"库存管理"页面
2. 找到库存不足的商品
3. 点击"调整库存"按钮
4. 选择"入库"
5. 输入库数量
6. 填写入库原因
7. 点击"确定"按钮
8. 系统提示调整成功

**方法二：设置库存预警**

1. 在商品列表中找到需要设置预警的商品
2. 点击"设置预警"按钮
3. 输入预警库存值
4. 点击"确定"按钮
5. 系统提示设置成功
6. 当库存低于预警值时，系统会自动通知

**注意事项**：

- 建议定期盘点库存
- 及时补充库存
- 避免库存积压

### Q13: 如何编辑商品信息？

**A**: 编辑商品信息的步骤如下：

1. 进入"商品列表"页面
2. 找到需要编辑的商品
3. 点击"编辑"按钮
4. 修改商品信息
5. 点击"保存"按钮
6. 系统提示保存成功

**编辑限制**：

- 商品ID不能修改
- 商品创建时间不能修改
- 商品销量不能修改（系统自动统计）

### Q14: 如何删除商品？

**A**: 删除商品的步骤如下：

1. 进入"商品列表"页面
2. 找到需要删除的商品
3. 点击"删除"按钮
4. 系统弹出确认对话框
5. 点击"确定"按钮
6. 系统提示删除成功

**删除限制**：

- 有未完成订单的商品不能删除
- 建议先下架商品，确认无订单后再删除

**建议**：

- 对于不再销售的商品，建议先下架而不是删除
- 下架的商品可以重新上架
- 删除的商品无法恢复

## 3. 订单管理问题

### Q15: 如何查看订单详情？

**A**: 查看订单详情的步骤如下：

1. 点击左侧导航菜单的"订单管理"
2. 进入"订单列表"页面
3. 找到需要查看的订单
4. 点击"查看详情"按钮
5. 进入"订单详情"页面
6. 显示订单完整信息

**订单详情包含**：

- 订单基本信息（订单号、创建时间、订单状态）
- 用户信息（用户昵称、手机号、地址）
- 商品信息（商品列表、数量、单价、小计）
- 金额信息（商品总额、优惠金额、实付金额）
- 支付信息（支付方式、支付时间、交易号）
- 配送信息（配送地址、配送时间、配送人员）
- 备注信息（订单备注、商家备注）

### Q16: 如何确认订单？

**A**: 确认订单的步骤如下：

1. 在订单列表中找到待确认的订单
2. 点击"确认订单"按钮
3. 系统弹出确认对话框
4. 填写商家备注（可选）
5. 点击"确定"按钮
6. 系统提示确认成功
7. 订单状态变为"待配送"

**注意事项**：

- 确认前请仔细核对订单信息
- 确认后订单将进入配送流程
- 如有异常情况，请联系客户

**建议**：

- 及时确认订单（建议10分钟内）
- 发现问题及时联系客户

### Q17: 如何取消订单？

**A**: 取消订单的步骤如下：

1. 在订单列表中找到需要取消的订单
2. 点击"取消订单"按钮
3. 系统弹出取消对话框
4. 选择取消原因：
   - 用户主动取消
   - 商品缺货
   - 配送范围外
   - 其他原因
5. 填写取消说明（可选）
6. 点击"确定"按钮
7. 系统提示取消成功
8. 订单状态变为"已取消"

**取消限制**：

- 已支付的订单需要先退款才能取消
- 配送中的订单不能取消
- 已完成的订单不能取消

### Q18: 如何申请退款？

**A**: 申请退款的步骤如下：

1. 在订单列表中找到需要退款的订单
2. 点击"申请退款"按钮
3. 系统弹出退款对话框
4. 选择退款类型：
   - 全额退款
   - 部分退款
5. 填写退款金额
6. 选择退款原因：
   - 商品问题
   - 配送问题
   - 服务问题
   - 其他原因
7. 填写退款说明（可选）
8. 点击"提交"按钮
9. 系统提示提交成功
10. 等待审核（1-3个工作日）

**退款审核流程**：

1. 用户提交退款申请
2. 商家审核退款申请
3. 审核通过后，系统发起退款
4. 支付平台处理退款
5. 退款到账（1-3个工作日）
6. 系统通知用户退款结果

**退款状态**：

- 待审核：退款申请已提交，等待审核
- 审核通过：退款申请审核通过，等待退款
- 退款中：退款正在处理中
- 退款成功：退款已到账
- 退款失败：退款失败，请联系客服
- 审核拒绝：退款申请被拒绝

### Q19: 如何安排配送？

**A**: 安排配送的步骤如下：

1. 在订单列表中找到待配送的订单
2. 点击"安排配送"按钮
3. 系统弹出配送对话框
4. 选择配送人员
5. 设置预计配送时间
6. 点击"确定"按钮
7. 系统提示安排成功
8. 订单状态变为"配送中"

**配送人员选择**：

- 从配送人员列表中选择
- 可以查看配送人员的工作状态
- 可以查看配送人员的配送记录

**预计配送时间**：

- 根据配送距离和路况估算
- 建议预留充足时间
- 可以根据实际情况调整

### Q20: 如何导出订单？

**A**: 导出订单的步骤如下：

1. 在订单列表页面
2. 设置筛选条件（可选）
3. 点击"导出"按钮
4. 选择导出格式：

5. 点击"确定"按钮
6. 系统生成文件

**导出内容**：

- 订单基本信息
- 用户信息
- 商品信息

**注意事项**：

- 一次最多导出10000条订单
- 导出时间取决于数据量
- 建议在非高峰时段导出

## 4. 支付相关问题

### Q21: 支持哪些支付方式？

**A**: 平台支持以下支付方式：

| 支付方式     | 说明                         | 手续费 |
| ------------ | ---------------------------- | ------ |
| **微信支付** | 扫码支付、H5支付、小程序支付 | 0.6%   |
| **支付宝**   | 扫码支付、H5支付、APP支付    | 0.6%   |
| **银行卡**   | 信用卡、借记卡               | 0.5%   |
| **余额支付** | 使用账户余额支付             | 0%     |

**支付方式配置**：

1. 进入"支付管理"页面
2. 点击"支付方式"子菜单
3. 选择需要配置的支付方式
4. 点击"配置"按钮
5. 填写配置信息
6. 点击"测试连接"
7. 测试成功后点击"保存"

### Q22: 支付失败怎么办？

**A**: 支付失败的常见原因及解决方法：

**原因一：账户余额不足**

- 解决方法：充值或选择其他支付方式

**原因二：网络连接问题**

- 解决方法：检查网络连接，重新支付

**原因三：支付方式未配置**

- 解决方法：联系商家配置支付方式

```typescript

// ✅ 正确：清理事件监听器
useEffect(() => {
  window.addEventListener("resize", handleResize);
  return () => {
    window.removeEventListener("resize", handleResize);
  };
}, []);
```

```typescript
// ❌ 错误：每次都创建新对象

const memoizedData = useMemo(() => ({ value: 1 }), []);
```

```typescript
// ❌ 错误：传递整个大对象
```

```typescript
  constructor(
    public statusCode: number,
    public message: string,
    public isOperational = true
  ) {
    super(message);
    this.name = this.constructor.name;
    Error.captureStackTrace(this, this.constructor);
  }
}

app.use((err: Error, req: Request, res: Response, next: NextFunction) => {

      success: false,
      error: err.message,
```
