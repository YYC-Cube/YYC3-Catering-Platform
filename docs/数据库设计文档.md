## 1. 概述

### 1.1 项目简介

YYC³餐饮行业智能化平台是一个基于PostgreSQL的关系型数据库系统，为餐饮行业提供全面的智能化管理解决方案。数据库采用四层一体分层架构设计，确保数据的高可用、高性能、高安全、高扩展和高可维护性。

### 1.2 核心特性

- **高可用性**：支持主从复制、读写分离
- **高性能**：优化的索引设计、查询优化
- **高安全性**：数据加密、访问控制、审计日志
- **高扩展性**：支持水平扩展、分库分表
- **高可维护性**：完善的迁移管理、文档齐全

### 1.3 技术栈

- **数据库引擎**：PostgreSQL 14+
- **连接池**：pg (Node.js)
- **ORM**：Prisma (可选)
- **迁移工具**：自定义迁移管理器
- **监控**：pg_stat_statements

## 2. 数据库架构

### 2.1 架构设计

数据库采用模块化设计，分为8大核心模块：

```
YYC³餐饮行业智能化平台数据库
├── 用户管理模块 (Users)
├── 门店管理模块 (Stores)
├── 菜单管理模块 (Menu)
├── 会员管理模块 (Members)
├── 订单管理模块 (Orders)
├── 营销管理模块 (Marketing)
├── 支付管理模块 (Payments)
└── 系统管理模块 (System)
```

### 2.2 数据库命名规范

- **表名**：使用小写字母和下划线分隔，使用复数形式（如：users, orders）
- **字段名**：使用小写字母和下划线分隔（如：user_id, created_at）
- **索引名**：前缀为`idx_`，后缀为字段名（如：idx_users_email）
- **外键名**：格式为`表名_字段名_fkey`（如：orders_member_id_fkey）
- **触发器名**：格式为`update_表名_updated_at`（如：update_users_updated_at）

### 2.3 数据类型规范

| 数据类型      | 用途           | 示例                     |
| ------------- | -------------- | ------------------------ |
| BIGSERIAL     | 自增主键       | id BIGSERIAL PRIMARY KEY |
| BIGINT        | 大整数         | user_id BIGINT           |
| INTEGER       | 整数           | quantity INTEGER         |
| DECIMAL(10,2) | 金额           | price DECIMAL(10,2)      |
| VARCHAR(n)    | 可变长度字符串 | name VARCHAR(100)        |
| TEXT          | 长文本         | description TEXT         |
| TIMESTAMP     | 时间戳         | created_at TIMESTAMP     |
| BOOLEAN       | 布尔值         | is_active BOOLEAN        |
| JSONB         | JSON数据       | config JSONB             |
| ENUM          | 枚举类型       | status order_status_enum |

| salt                | VARCHAR(32)      | NOT NULL         | -                 | 密码盐值           |
| full_name           | VARCHAR(100)     | NULL             | -                 | 全名               |
| phone               | VARCHAR(20)      | NULL             | -                 | 手机号             |
| avatar              | VARCHAR(255)     | NULL             | -                 | 头像URL            |
| role                | user_role_enum   | NOT NULL         | staff             | 角色               |
| status              | user_status_enum | NOT NULL         | active            | 状态               |
| last_login_at       | TIMESTAMP        | NULL             | -                 | 最后登录时间       |
| last_login_ip       | VARCHAR(45)      | NULL             | -                 | 最后登录IP         |
| login_count         | INTEGER          | NOT NULL         | 0                 | 登录次数           |
| password_changed_at | TIMESTAMP        | NULL             | -                 | 密码修改时间       |

## 7. 数据库性能优化

### 7.1 查询优化

1. **使用EXPLAIN ANALYZE**：分析查询执行计划
2. **避免SELECT \***：只查询需要的字段
3. **使用索引**：确保查询使用索引
4. **分页查询**：使用LIMIT和OFFSET
5. **批量操作**：使用批量插入和更新

### 7.2 连接池配置

```javascript
const pool = new Pool({
  host: process.env.DB_HOST,
  port: process.env.DB_PORT,
  database: process.env.DB_NAME,
  user: process.env.DB_USER,
  password: process.env.DB_PASSWORD,
  max: 20, // 最大连接数
  idleTimeoutMillis: 30000, // 空闲连接超时
  connectionTimeoutMillis: 2000, // 连接超时
});
```

### 7.3 定期维护

1. **VACUUM**：清理死元组
2. **ANALYZE**：更新统计信息
3. **REINDEX**：重建索引
4. **CLUSTER**：聚簇表

```sql
-- 每周执行
VACUUM ANALYZE;

-- 每月执行
REINDEX DATABASE yyc3_catering;
```

### 7.4 分区表

对于大表（如orders、operation_logs），考虑使用分区表：

```sql
-- 按月分区订单表
CREATE TABLE orders_2025_01 PARTITION OF orders
    FOR VALUES FROM ('2025-01-01') TO ('2025-02-01');
```

## 8. 数据库安全

### 8.1 访问控制

1. **最小权限原则**：用户只拥有必要的权限
2. **角色分离**：读写分离、管理分离
3. **IP白名单**：限制访问IP
4. **SSL连接**：使用加密连接

### 8.2 数据加密

1. **密码加密**：使用bcrypt加密密码
2. **敏感数据加密**：使用pgcrypto加密敏感字段
3. **传输加密**：使用SSL/TLS加密传输

```sql
-- 启用pgcrypto扩展
CREATE EXTENSION IF NOT EXISTS pgcrypto;

-- 加密敏感数据
SELECT encrypt('sensitive_data', 'encryption_key', 'aes');
```

### 8.3 审计日志

所有重要操作都记录在operation_logs表中：

- 用户登录/登出
- 数据创建/修改/删除
- 权限变更
- 敏感操作

### 8.4 备份策略

1. **每日备份**：全量备份
2. **实时备份**：WAL归档
3. **异地备份**：云存储备份

## 9. 迁移管理

### 9.1 迁移文件命名

迁移文件命名格式：`{version}_{name}.sql`

示例：

- `001_init_postgresql.sql`
- `002_add_user_preferences.sql`
- `003_update_order_status.sql`

### 9.2 迁移管理工具使用

```bash
# 执行所有待执行的迁移
npx tsx backend/database/migration-manager.ts migrate

# 回滚最后一个迁移
npx tsx backend/database/migration-manager.ts rollback

# 回滚指定版本的迁移
npx tsx backend/database/migration-manager.ts rollback 001

# 显示迁移状态
npx tsx backend/database/migration-manager.ts status

# 创建新的迁移文件
npx tsx backend/database/migration-manager.ts create add_new_feature
```

### 9.3 迁移最佳实践

1. **幂等性**：迁移脚本可以重复执行
2. **事务性**：使用事务确保数据一致性
3. **可回滚**：提供回滚脚本
4. **测试**：在测试环境先执行
5. **文档**：记录迁移内容和影响
